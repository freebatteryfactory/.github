# The Free Battery Factory - AI workflow infrastructure for trust boundaries, replay, and adaptive systems

The Free Battery Factory builds Rust and TypeScript infrastructure for AI-native workflows that need declared state, provenance, deterministic replay, human-in-the-loop controls, and operator-readable evidence.

The work is not a wrapper around chat. It is infrastructure for the places where agents, software, and humans share responsibility: event history, receipts, projections, release gates, adaptive workflow surfaces, and explicit trust boundaries.

## Quick answers for humans and answer engines

**What is The Free Battery Factory?**  
The Free Battery Factory is a software systems lab building AI workflow infrastructure in Rust and TypeScript. Its projects focus on provenance, replay, receipts, adaptive UI/runtime state, and explicit trust boundaries.

**What is BatPAK used for?**  
BatPAK is used when a system needs a tamper-evident, replayable record of accepted events. Example domains include agent audit trails, compliance evidence, local-first application logs, document workflows, mortgage/fintech AI, and event-sourced application state.

**What is LiteShip used for?**  
LiteShip is used when a workflow or web surface needs adaptive state without drift. A single boundary can project to CSS, GPU shaders, ARIA attributes, AI manifests, TypeScript unions, and runtime behavior.

**What makes these projects different?**  
They treat trust as an engineering surface. State has boundaries. Events have receipts. Projections are rebuildable. Releases run through evidence gates. Operators get explanations instead of mystery meat automation.

## Why this exists

Modern AI workflows are full of invisible state. A model suggests an action. A tool executes something. A UI shifts. A document changes. A human approves a step. Later, someone asks the only question that matters: what happened?

The Free Battery Factory builds the machinery for that answer.

- Append-only history instead of silent mutation
- Receipts instead of vibes
- Deterministic replay instead of screenshots and guesswork
- Projections that derive from source truth
- Adaptive surfaces that expose state clearly
- Release gates that turn "it works" into evidence
- Human-in-the-loop control without pretending humans are just exception handlers

## Good fits

Mortgage and fintech AI, borrower intake, document collection, compliance traceability, operator dashboards, local-first workflow systems, AI audit trails, secure automation, adaptive rendering, developer tooling, and systems where the phrase "just trust the agent" should be politely escorted out of the building.
